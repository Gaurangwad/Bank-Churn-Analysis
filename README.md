# Bank Churn Analysis — Customer Retention Intelligence Dashboard

An interactive dashboard that analyses bank customer churn, predicts which
customers are at risk of leaving, and surfaces the metrics retention teams
actually need. It ships with a built-in demo dataset and lets you upload your
own CSV to get real results.

## Features

- **Two data sources**
  - **Demo dataset** — 1,000 realistic synthetic customers. The prediction
    model scores **100% accuracy** on this set by construction (labels are
    generated by the same model used to predict them), so the dashboard always
    has a clean showcase.
  - **Upload your own CSV** — drag & drop or browse. Works out of the box with
    the standard Kaggle `Churn_Modelling.csv` schema (column matching is
    case/spacing-insensitive and supports common aliases). Uploaded data is
    scored with the real model and produces **genuine** accuracy, precision,
    recall, F1, specificity, ROC-AUC and a confusion matrix.
- **KPI cards** — total customers, churn rate, retained, high-risk count,
  average balance/credit score, and estimated revenue at risk.
- **Model performance panel** — full classification metrics plus a confusion
  matrix, evaluated against the ground-truth `Exited` label. If an uploaded
  file has no label column, the app says so and shows predictions only.
- **Interactive charts** (Recharts) — churn by geography, age group, products
  held, membership activity and gender, plus the predicted risk distribution.
- **Customer risk register** — every customer ranked by churn probability with
  search, high-risk filtering and pagination.
- **100% local** — all parsing and scoring runs in the browser; uploaded data
  never leaves the device.

## The prediction model

A deterministic, explainable logistic risk model (`src/lib/churn.ts`) built on
the churn drivers established in the public bank-churn dataset: customer age,
membership activity, number of products held, geography (Germany churns ~2×),
gender, balance, credit score and tenure. Because it is a pure function of the
feature vector, it is fully reproducible and auditable.

## Upload format

Provide a CSV with these columns (a downloadable template is available in the
app via **Data Source → Template**):

```
CustomerId, Surname, CreditScore, Geography, Gender, Age, Tenure,
Balance, NumOfProducts, HasCrCard, IsActiveMember, EstimatedSalary, Exited
```

`Exited` (or `Churn`/`Churned`) is optional — include it to measure model
accuracy, omit it to get predictions only.
